# 🚀 Инструкция по настройке Telegram бота для приёма заявок

## Шаг 1: Создай Telegram бота

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Напиши команду `/newbot`
3. Придумай имя бота (например: `monkedDev Orders`)
4. Придумай юзернейм бота (обязательно заканчивается на `bot`, например: `monkeddev_orders_bot`)
5. **Скопируй токен** (выглядит как `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

---

## Шаг 2: Узнай свой Chat ID

1. Открой [@userinfobot](https://t.me/userinfobot)
2. Нажми `/start`
3. **Скопируй свой Id** (число, например: `123456789`)

---

## Шаг 3: Задеплой сайт на Vercel

### Вариант А: Через Vercel Dashboard (проще)

1. Зайди на [vercel.com](https://vercel.com)
2. Войди через GitHub
3. Нажми **Add New Project**
4. Выбери репозиторий `monkedd-develop`
5. Нажми **Import**
6. В настройках проекта добавь **Environment Variables**:
   - `TELEGRAM_BOT_TOKEN` = твой токен от BotFather
   - `TELEGRAM_CHAT_ID` = твой Chat ID
7. Нажми **Deploy**

### Вариант Б: Через CLI

```bash
# Установи Vercel CLI
npm i -g vercel

# Залогинься
vercel login

# Задеплой
vercel --prod
```

После деплоя добавь переменные окружения в настройках проекта на Vercel.

---

## Шаг 4: Проверь работу

1. Открой задеплоенный сайт на Vercel
2. Заполни форму заказа
3. Проверь Telegram — должно прийти сообщение!

## 🔒 Безопасность

- ✅ Токен бота хранится на сервере Vercel (не виден в коде)
- ✅ CORS защищает от запросов с других сайтов
- ✅ Honeypot поле ловит спам-ботов
- ✅ Валидация данных на сервере

## ❓ Вопросы

**Q: Можно ли отправлять заявки ещё и на почту?**  
A: Да! В коде уже есть дублирование на Web3Forms — заявки будут приходить и в Telegram, и на почту.

**Q: Что если токен украдут?**  
A: С Vercel это не проблема — токен хранится в переменных окружения на сервере, его не видно в коде.

**Q: Можно ли изменить формат сообщения?**  
A: Да! Отредактируй `formattedMessage` в файле `api/send-telegram.js`.
